# Domain Password Reset Procedure

<Info>
**Quick Answer**: Reset domain passwords through Active Directory Users and Computers, then unlock account if necessary.
</Info>

## Problem Summary

### What You'll Experience
- User can't login with error "The username or password is incorrect"
- Account may be locked after multiple failed attempts  
- User reports forgotten password after time away

### Why This Happens
Domain passwords expire based on company policy, or users forget them after extended absences.

---

## Before You Start

<Warning>
**Required Access & Prerequisites**
- [ ] Administrative access to Active Directory
- [ ] User identity verification completed
- [ ] Confirm it's a domain account (not local)
- [ ] Note: Changes take effect immediately
</Warning>

---

## Solution

<Steps>
  <Step title="Open Active Directory Console">
    **Launch AD Management**: Press `Win + R`, type `dsa.msc`, press Enter
    
    **Expected Result**: Active Directory Users and Computers opens
    
    <Note>
    **If access denied**: Verify you have domain admin rights or delegated password reset permissions
    </Note>
  </Step>
  
  <Step title="Locate User Account">
    **Search for User**: Press `Ctrl + F` to open Find dialog
    
    **Search Steps**:
    - Enter username or full name
    - Click "Find Now"
    - Double-click user from results
    
    **Expected Result**: User account properties open
  </Step>
  
  <Step title="Reset Password">
    **Reset Process**:
    - Right-click user account → "Reset Password"
    - Enter secure temporary password (follows complexity policy)
    - ✅ Check "User must change password at next logon"
    - ✅ Check "Unlock the user's account" if needed
    - Click "OK"
    
    **Expected Result**: "The password for [username] has been changed" confirmation
  </Step>
</Steps>

<Check>
**Success Verification**

### Next Steps
1. **Communicate** temporary password securely to user
2. **Confirm** user can change password successfully  
3. **Verify** normal application access works

## Troubleshooting

<AccordionGroup>
  <Accordion title="Reset option is grayed out">
    **Cause**: Insufficient permissions for password reset
    
    **Solution**: 
    - Verify you have "Reset Password" rights in AD
    - Contact domain administrator for delegation
    - Check if account is in protected OU
  </Accordion>
  
  <Accordion title="User still can't login after reset">
    **Possible Causes**:
    - Account is disabled
    - Time/date synchronization issue
    - Group policy restriction
    
    **Solutions**:
    - Check Account tab → ensure account is enabled
    - Verify time sync between client and domain controller
    - Review applied group policies for logon restrictions
  </Accordion>
</AccordionGroup>
